## 目的

Yagura プロジェクト（以下「本プロジェクト」。メンテナ: YANAI Taketo、以下「メンテナ」）は Apache License 2.0 の下で公開されています。本契約は、あなた（以下「コントリビューター」）が本プロジェクトへ提供する貢献（Contribution）について、メンテナが本プロジェクトを Apache License 2.0 の下で継続的に配布・利用できるよう、必要な権利を明確にするものです。**本契約はあなたの著作権を移転しません**——あなたは自分の貢献の著作権を保持し続けます。本契約はライセンスの付与に留まります。

## 定義

- **「貢献（Contribution）」**: あなたが本プロジェクトへ提出（Submit）する、オリジナルの著作物（コード・ドキュメント・その他の素材）およびそれらへの変更・追加をいいます。
- **「提出（Submit）」**: 本プロジェクトのリポジトリ・課題管理・レビューシステム等を通じて、メンテナまたは本プロジェクトへ送付・投稿するあらゆる形態の電子的・書面・口頭のやり取りをいいます（ただし、あなたが明確に「貢献ではない（Not a Contribution）」と表示した通信を除きます）。

## 1. 著作権ライセンスの付与

あなたは、あなたの貢献について、メンテナおよび本プロジェクトの成果物の受領者に対し、**恒久的・全世界的・非独占的・無償・ロイヤリティ不要・取消不能**の著作権ライセンスを付与します。このライセンスには、あなたの貢献およびその二次的著作物を、複製・改変・展示・実演・サブライセンス・頒布する権利が含まれます。メンテナは、あなたの貢献を **Apache License 2.0（またはメンテナが将来採用する、これと互換の OSI 承認オープンソースライセンス）** の下で頒布できます。

## 2. 特許ライセンスの付与

あなたは、あなたの貢献について、メンテナおよび本プロジェクトの成果物の受領者に対し、**恒久的・全世界的・非独占的・無償・ロイヤリティ不要・取消不能**（本条に定める場合を除く）の特許ライセンスを付与します。対象は、あなたの貢献単体、または貢献と提出先の本プロジェクトの組み合わせによって必然的に侵害される、あなたがライセンス可能な請求項に限られます。あなた（またはあなたのために行動する者）が、本プロジェクトまたは貢献が特許を侵害すると主張する特許訴訟（交差請求・反訴を含む）を提起した場合、本契約に基づき当該相手方へ付与された特許ライセンスは、その提訴日をもって終了します（Apache License 2.0 第 3 条と同趣旨）。

## 3. 表明

あなたは次を表明します:

1. あなたは本契約を締結する法的権限を有し、上記のライセンスを付与する権利を有します。
2. 各貢献はあなたのオリジナルの創作物です（第 4 条に定める場合を除く）。
3. あなたの貢献には、あなたが認識している限り、あなたが本契約に基づき付与できない第三者のライセンス・その他の制限（特許・商標・企業秘密・ライセンス契約等）が付随しません。
4. **雇用主等がある場合**: あなたの貢献があなたの雇用の範囲で作成された、または雇用主が知的財産権を主張しうる場合、あなたは、雇用主から本契約の付与について許諾を得ているか、雇用主が当該貢献について権利放棄を行っていることを表明します。該当する場合は、法人向けの [企業コントリビューターライセンス契約（Corporate CLA）](cla-corporate.md) の手続きをオーナーと確認してください。

## 4. 第三者の著作物

あなたのオリジナルの創作物ではない著作物を提出する場合は、その出所・関連するライセンス・既知の制限に関する完全な詳細を、貢献とともに（例: 「Submitted on behalf of a third-party: [名称]」と明記して）別途示してください。

## 5. 保証の否認

あなたの貢献は、法律で要求される場合または書面で合意した場合を除き、**「現状有姿（AS IS）」** で提供され、明示黙示を問わずいかなる保証もありません（Apache License 2.0 第 7 条と同趣旨）。あなたは本プロジェクトへの技術サポートを提供する義務を負いません。

## 6. 情報の正確性の通知

上記の表明に不正確が生じたと気づいた場合は、メンテナへ通知してください。

## 署名（CLA Assistant による電子署名）

本プロジェクトは各 Pull Request 上で **CLA Assistant Lite**（自己ホスト型 GitHub Action）により署名を受け付けます。あなたが自分の PR に次の定型文をコメントすることで、本契約に同意し署名したものとみなされます:

> **I have read the CLA Document and I hereby sign the CLA**

（＝「私は CLA 文書を読み、ここに CLA へ署名します」）

署名（GitHub ユーザー名）は本リポジトリ内の署名台帳ファイルに記録されます。第三者のホスト型サービスへは送信されません。
