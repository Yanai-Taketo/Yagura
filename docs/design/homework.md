# 全体設計書の宿題一覧

ADR 0001〜0006 の本文とペルソナレビュー（PR #1・#3・#5・#7・#9・#11 のコメント記録）で全体設計書・後続文書に委任された事項の集約。**全体設計書の各文書を起筆する際は、まずこの一覧から該当項目を引き取り、解決した項目には解決先（文書・節）を記入する**。全項目が解決先を持ったらこの文書の役目は終わる。

凡例: ⬜ 未解決 / 出典は ADR 番号（本文）または PR 番号（レビュー申し送り）

## architecture.md（受信パイプライン・信頼性・性能）

- ⬜ 内部バッファの容量・バッチサイズ等の数値をベンチ実測で確定（ADR-0002）——機構は [architecture.md](architecture.md) §2.1・§3.1 で確定済み、数値は同 §9 M-1 で追跡
- ✅ 解決 → [architecture.md](architecture.md) §3.2.3（新規破棄を採用。破棄は計上）: スプール上限到達時の挙動（ADR-0002）
- ✅ 解決 → [architecture.md](architecture.md) §3.2.4（通常削除で速やかに削除。上書き消去は約束せず限界を利用者向けに明記）: スプールの削除保証（ADR-0004 決定 5）
- ✅ 解決 → [architecture.md](architecture.md) §3.2.2（ライブ優先ヒステリシス + 速度上限。水位・上限値は §9 M-3）: スプール drain の流量制御（PR #3）
- ✅ 解決 → [architecture.md](architecture.md) §3.2.5（定期自己検証 + drain 書込コードのライブ経路共有）: スプール経路の平常時検証（ADR-0002 却下案 (c) の代償措置）
- ⬜ 流量制御の既定閾値（「通常運用で発火しない安全側」を実測で）と提供時期（初期バージョン群）（ADR-0002）——挿入点の位置は [architecture.md](architecture.md) §3.3 で確定済み、閾値は同 §9 M-4 で追跡
- ⬜ OS レベル取りこぼし観測の実現手段と粒度（実機検証。意図粒度で不可ならシステム全体統計へ縮退 + ADR-0002 amendment）（ADR-0002）——方針と突合方式は [architecture.md](architecture.md) §4.2 で確定済み、手段の確定は同 §9 M-8 で追跡
- ✅ 解決 → [architecture.md](architecture.md) §4.3（定期永続化 + 正常停止時の書き切り。クラッシュ窓は間隔以下に限定）: ドロップカウンタの再起動をまたぐ永続化（PR #3）
- ✅ 解決 → [architecture.md](architecture.md) §4.4（停止イベント + 生存時刻による受信断区間の記録・時間軸表示）: 再起動由来の受信断・欠損の可視化（PR #3・#5）
- ✅ 解決 → [architecture.md](architecture.md) §4.5（UDP はサーバ到達後のみ観測可 / TCP は接続事象を計測、送信側依存を明記）: 受信プロトコル別のロス特性の精緻化（PR #5 クリス）
- ⬜ ベンチ合否の判定方式 + **性能公称値の確定**（ADR-0002 / ADR-0006 基準 1）——判定方式は [architecture.md](architecture.md) §5.2 で確定済み（CI 回帰 = 基準比 / 公称 = 基準環境の絶対値）、公称値は同 §9 M-6 で追跡（**引き下げは supersession 準拠** — ADR-0006 運用）
- ⬜ SQLite の実用限界の実測、飽和の観測可能化、昇格案内の発火条件（ADR-0002）——観測機構と発火条件の形は [architecture.md](architecture.md) §5.3 で確定済み（シグナル = スプール退避カウンタ）、閾値は同 §9 M-7 で追跡
- ✅ 解決 → [architecture.md](architecture.md) §6（絞り込み強制なし + 結果上限 + 軽量射影 + タイムアウト。既定値は仮値、§9 M-10 で追跡）: UI の重い操作の防御（ADR-0002 帰結 / ADR-0003）
- ⬜ サービス再起動時間の最小化目標（PR #3）——最小化の設計（受信先行起動・スプール退避停止）は [architecture.md](architecture.md) §1.2・§1.3 で確定済み、目標値は同 §9 M-9 で追跡

## セキュリティ設計（ADR-0004 の詳細化。置き場は起筆時に判断）

- ⬜ loopback 束縛の CI 回帰テスト（127.0.0.1/::1 両系 + WebSocket 昇格後の維持。**v0.1 受け入れ条件**）（ADR-0004 決定 2）
- ⬜ circuit 上限の既定値・単位・到達時挙動（障害時の閲覧集中を入力に。既存・新規のどちらを守るか + 到達時の通知経路）（ADR-0004 決定 6 / PR #7 鈴木）
- ⬜ 掲示用途（常時表示ダッシュボード）を認証失効で不意に切らない方式（ADR-0004 決定 6）
- ⬜ 役割の粒度（最小: 閲覧/管理の 2 段）（ADR-0004 決定 3）
- ⬜ 監査記録の出力先・保持期間 + Windows イベントログの**イベント ID 採番方針の早期固定** + 拒否試行の流量制御（証跡の希釈・ディスク圧迫対策） + 記録失敗の通知経路（イベントログ自体が死んでいる場合を含む）（ADR-0004 決定 7 / PR #7 鈴木・田中）
- ⬜ スプール・組み込み DB・設定ファイルの ACL 具体値（ADR-0004 決定 5）
